### Задача (Теория): Что такое CI (Continuous Integration)? ###
CI (Continuous Integration / Непрерывная интеграция) – методология и набор практик в разработке, при которых изменения кода с заданной периодичностью автоматически объединяются в общий репозиторий и проходят серию автоматических проверок.<br>
CI-сервер можно настроить на перехват какого-либо события, после чего запускается пайплайн (pipeline) – последовательность шагов:<br>
* Сборка (Build) – компиляция кода, установка зависимостей.
* Статический анализ (Linting) – проверка кода на синтаксические ошибки и соответствие стандартам.
* Тестирование (Testing) – автоматические тесты для проверки работоспособности и влияния изменений на заданные области приложения.

Задача DevOps – описать конфигурацию CI:
* Где запускать проверки.
* Какие команды выполнять.
* Какие артефакты сохранять после успешной сборки (для дальнейшей передачи в деплой).

### Задача (Теория): Что такое CD (Continuous Delivery/Deployment)? ###
CD делится на две принципиально разные практики:
* Continuous Delivery (непрерывная доставка) – код, прошедший CI, формируется в релиз, готовый к развертыванию в продакшене. Для развертывания релиза необходимо решение бизнеса. Это дает бизнесу дополнительный контроль.
* Continuous Deployment (непрерывное развертывание) – код, прошедший CI, формируется в релиз, который автоматически развертывается в продакшене, без вмешательства бизнеса.

### Задача (Теория): Анатомия GitHub Actions (или GitLab CI). ###
Workflow (Воркфлоу / Пайплайн) – задается в YAML-файле, который лежит в репозитории по пути:
```
.github/workflows/имя_файла.yml
```
Описывает что, когда и где должно произойти.<br>
```name:``` - параметр для задания имени воркфлоу.

Event (Триггер / Событие) – условие запуска воркфлоу.<br>
```on:``` - параметр для задания условия запуска воркфлоу.

Runner (Раннер) – физическая или виртуальная машина, на которой будут исполняться команды.<br>
```runs-on:``` - параметр для задания раннера. Прописывается внутри джобы.

Job (Джоб / Задача) – логическая группа шагов для выполнения на заданном раннере. По умолчанию джобы выполняются параллельно.<br>
```needs:``` - параметр для установки очередности выполнения джобов.<br>
Имена джобов задаются произвольно.<br>
```jobs:``` - параметр для обозначения начала описания джобов.

Step (Степ / Шаг) – конкретное действие. Шаги внутри джобы выполняются последовательно.<br>
```steps:``` - параметр для обозначения начала описания шагов.<br>
Виды шагов:<br>
```run:``` – выполнение команды в терминале.<br>
```uses:``` - вызов готовой «Action».
