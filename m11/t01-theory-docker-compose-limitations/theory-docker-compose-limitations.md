«Docker Compose» (DC) подходит для локальной разработки, CI/CD пайплайнов и проектов на одном сервере. Отсутствие полноценной оркестрации делает неудобным его использование в продакшене с высокими требованиями к доступности.

Проблемы:
* Самовосстановление (```self-healing```):<br>
«DC» может перезапустить контейнер (```restart: always```), но только на уровне одного хоста.<br>
Если выйдет из строя сервер, «DC» не сможет перенести сервисы на другой, рабочий сервер.<br>
«DC» не умеет выполнять сложные проверки работоспособности контейнера с автоматической заменой в случае возникновения проблем.

* Масштабирование и балансировка (```scaling```):<br>
«DC» может выполнить масштабирование (```--scale```) только в пределах одного хоста. Такое масштабирование будет ограничено его ресурсами.<br>
У «DC» нет встроенного балансировщика трафика. Необходимо настраивать внешний балансировщик (Nginx, HAProxy и т.д.) и каждый раз менять его конфигурацию при изменении количества реплик.

* Развертывание без простоя (```zero-downtime deployment```):<br>
При обновлении «DC» сначала остановит старые контейнеры, а затем запустит новые.<br>
Если контейнер с новой версией образа не запустился, «DC» не вернет старую версию образа автоматически.
