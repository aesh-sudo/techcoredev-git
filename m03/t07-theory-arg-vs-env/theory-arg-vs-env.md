```ARG``` – переменные, значения которых доступны только во время сборки образа.<br>
```ARG```, объявленный до ```FROM```, доступен только во ```FROM```.<br>
Можно переопределить при ```docker build``` с помощью параметра ```--build-arg```.<br>

```ENV``` – переменные, значения которых будут доступны во время сборки и в командной оболочке приложения.<br>
```-e``` – определение/переопределение переменных при запуске контейнера.<br>

Связка ```ARG``` и ```ENV``` – используется, чтобы передать в контейнер данные, которые указываются при сборке. Сначала данные записываются в ```ARG```, а затем значение ```ARG``` помещается в ```ENV```.<br>
Пример:<br>
dockerfile:
```
FROM alpine

# 1. Принимаем значение при сборке
ARG APP_VERSION=1.0.0

# 2. Записываем его в переменную окружения, которая останется в образе
ENV APP_VERSION=${APP_VERSION}

# При запуске контейнера приложение сможет прочитать $APP_VERSION
CMD ["sh", "-c", "echo Запущена версия $APP_VERSION"]
```
Сборка:
```bash
docker build --build-arg APP_VERSION=2.5.4 -t myapp .
```
