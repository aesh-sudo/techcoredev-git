«Multi-stage build» применяется в основном для приложений, написанных на компилируемых ЯП. Для сборки такого приложения требуются компилятор, сборщик, зависимости и т.д. Если выполнять сборку в одном слое, то, кроме большого веса, в образ попадут уязвимые инструменты сборки и компиляции, исходный код, кеши и т.д.<br>
Этап 1 – создание скомпилированного файла приложения.
```
FROM <image> AS <name>
```
На данном этапе скачиваются зависимости, компилируется код, создаются временные файлы и т.д. Все необходимые данные для создания приложения.<br>
Этап 2 – извлечение из этапа 1 только необходимых данных.
```
COPY –from=<name> <src> <dst>
```
При указании ```src``` и ```dst``` можно использовать шаблоны. Также можно использовать несколько инструкций:
```
COPY –from=<name>...
```
для копирования не связанных элементов.<br>
В финальный образ попадает только то, что было скопировано с помощью инструкций:
```
COPY –from=<name>...
```

Пример приложения, написанного на «Go»:
```go
FROM golang:1.20-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o myapp main.go

# Используем 'scratch' (пустой образ, 0 МБ) или alpine
FROM scratch
COPY --from=builder /app/myapp /myapp
CMD ["/myapp"]
```
